# Atlas Rover Mk.1 项目进展日志

这个日志记录项目从草图、采购、机械、电路到固件的一步步推进。它不会只写“成功”，也会记录踩坑和回头修正。

## 2026-06-16

### 完成

- 整理 Atlas Rover Mk.1 制造图纸包 V1.0。
- 明确 DualEye 不直接驱动电机，采用“双板方案”：
  - DualEye：语音、双目、页面、触摸、miniClaw/MimiClaw。
  - 底盘板：电机闭环、限速、超时停车、DRV8833。
- 确认 DualEye 官方资料：
  - 双 1.28 英寸 LCD，每屏 240 x 240。
  - 支持 Wi-Fi/BLE、I2C touch、音频、外露 UART/I2C/IO。
  - LCD1 Pin10 为 UART_TXD，Pin9 为 UART_RXD。
- 完成双目表情设计方案 V0.1：
  - 每块实体屏幕只显示一只眼睛。
  - 支持待机、开心、聆听、思考、说话、移动、好奇、困倦、惊讶、眨眼、拒绝、充电、错误。
- 完成 Web 表情预览：
  - 可在 VS Code Live Preview 中查看。
  - 底盘方向按钮会改变眼睛目光方向并输出 UART 指令日志。
- 完成 DualEye 固件 V0.1：
  - 表情参数模型。
  - UI 状态机。
  - UART `AR1,` 底盘协议。
  - 语音事件入口。
  - 700 ms DualEye 主动安全 STOP。
  - ESP-IDF 本地构建通过。
- 按 DualEye 官方规格把固件 Flash 配置改为 16MB。
- 整理 GitHub 仓库首版 README、贡献说明、路线图和同步脚本。
- 明确配网、API 与控制端方案：
  - DualEye 和底盘板按硬件边界分别烧录。
  - 端侧 MimiClaw 才与 DualEye 固件一起编译烧录；电脑宿主 MiniClaw 不烧进板子。
  - 首次配网走 SoftAP `AtlasRover-XXXX` / `192.168.4.1`。
  - 入网后用 Web 管理界面配置 LLM 模式、Base URL、Model、API Key。
  - 手机/电脑端统一先做响应式 Web 管理界面，不先做原生 App。
- 完成 DualEye 固件 V0.2 骨架：
  - 新增 NVS 配置模块 `atlas_config.*`。
  - 新增 SoftAP/STA/APSTA 网络模块 `atlas_wifi.*`。
  - 新增 Web 管理页和 REST API `atlas_admin_http.*`。
  - 新增 6 位本地配对码 `atlas_pairing.*`。
  - 新增 LLM 配置状态封装 `atlas_llm_client.*`。
  - 新增 MimiClaw/MiniClaw 文本意图适配层 `atlas_mimiclaw_adapter.*`。
  - STOP 不需要配对码，移动和配置修改需要配对码。
  - 默认 `motion_enabled=true`，开箱即可做短时移动测试；仍可在安全设置中关闭。
  - 增加 `control_mode`：默认 `manual` 支持 Web 手动控制，切到 `ai` 后只允许语音/MimiClaw 运动意图。
- 为 DualEye 固件增加自定义分区表：
  - 16MB Flash。
  - 4MB factory app 分区。
  - 预留 4MB storage 分区。
- V0.2 本地 `idf.py build` 编译通过。

### 仍然不确定

- Waveshare 官方双屏 ESP-IDF/LVGL 示例尚未接入到 `atlas_display.c`。
- PSRAM 模式暂未启用，需要跟官方显示示例一起验证。
- miniClaw/MimiClaw 语音链路还只是接口层，真实 LLM/宿主调用尚未实现。
- Web 管理页还只是基础嵌入式页面，没有做漂亮 UI、日志面板和 OTA。
- mDNS 暂未接入，当前依赖 SoftAP 地址或串口日志里的局域网 IP。
- API Key 当前存 NVS，但还未启用 NVS 加密，暂不适合放高价值密钥。
- 底盘板固件尚未开始。
- 铜丝车架需要真实打样后验证强度、绝缘和可维护性。

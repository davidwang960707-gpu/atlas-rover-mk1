# Atlas Rover Mk.1 项目进展日志

这个日志记录项目从草图、采购、机械、电路到固件的一步步推进。它不会只写“成功”，也会记录踩坑和回头修正。

## 2026-06-16

### 完成

- 整理 Atlas Rover Mk.1 制造图纸包 V1.0。
- 明确 DualEye 不直接驱动电机，采用“双板方案”：
  - DualEye：语音、双目、页面、触摸、MimiClaw。
  - 底盘板：普通 N20 + DRV8833 开环短时差速、限速、超时停车。
- 确认 DualEye 官方资料：
  - 双 1.28 英寸 LCD，每屏 240 x 240。
  - 支持 Wi-Fi/BLE、I2C touch、音频、外露 UART/I2C/IO。
  - LCD1 Pin10 为 UART_TXD，Pin9 为 UART_RXD。
- 完成双目表情设计方案 V0.1：
  - 每块实体屏幕只显示一只眼睛。
  - 支持待机、开心、聆听、思考、说话、移动、好奇、困倦、惊讶、眨眼、拒绝、充电、错误。
- 完成 Web 表情预览：
  - 可在 VS Code Live Preview 中查看。
  - 底盘方向按钮会改变眼睛目光方向并输出 UART 指令日志。
- 完成 DualEye 固件 V0.1：
  - 表情参数模型。
  - UI 状态机。
  - UART `AR1,` 底盘协议。
  - 语音事件入口。
  - 700 ms DualEye 主动安全 STOP。
  - ESP-IDF 本地构建通过。
- 按 DualEye 官方规格把固件 Flash 配置改为 16MB。
- 整理 GitHub 仓库首版 README、贡献说明、路线图和同步脚本。
- 明确配网、API 与控制端方案：
  - DualEye 和底盘板按硬件边界分别烧录。
  - 端侧 MimiClaw 才与 DualEye 固件一起编译烧录；外部宿主/调试桥不烧进板子。
  - 首次配网走 SoftAP `AtlasRover-XXXX` / `192.168.4.1`。
  - 入网后用 Web 管理界面配置 LLM 模式、Base URL、Model、API Key。
  - 手机/电脑端统一先做响应式 Web 管理界面，不先做原生 App。
- 完成 DualEye 固件 V0.2 骨架：
  - 新增 NVS 配置模块 `atlas_config.*`。
  - 新增 SoftAP/STA/APSTA 网络模块 `atlas_wifi.*`。
  - 新增 Web 管理页和 REST API `atlas_admin_http.*`。
  - 新增 6 位本地配对码 `atlas_pairing.*`。
  - 新增 LLM 配置状态封装 `atlas_llm_client.*`。
  - 新增 MimiClaw 文本意图适配层 `atlas_mimiclaw_adapter.*`。
  - STOP 不需要配对码，移动和配置修改需要配对码。
  - 默认 `motion_enabled=true`，开箱即可做短时移动测试；仍可在安全设置中关闭。
  - 增加 `control_mode`：默认 `manual` 支持 Web 手动控制，切到 `ai` 后只允许语音/MimiClaw 运动意图。
  - 拆分 Web 入口：`/app` 作为日常机器人应用控制页，`/admin` 作为低频管理后台。
- 为 DualEye 固件增加自定义分区表：
  - 16MB Flash。
  - 4MB factory app 分区。
  - 预留 4MB storage 分区。
- V0.2 本地 `idf.py build` 编译通过。
- 研读 `memovai/mimiclaw` README、中文 README、架构文档和 tool/skills 目录，纠正“MiniClaw 代替 MimiClaw”的错误口径。
- 完成 DualEye 固件 V0.5 MimiClaw 接口层：
  - 新增 `atlas_mimiclaw_intent.*`。
  - 新增 `/api/mimiclaw/intent`。
  - 管理后台新增 MimiClaw 结构化意图测试框。
  - 支持 Mimiclaw tool-call 风格 JSON，如 `atlas_set_expression`、`atlas_show_page`、`atlas_rover_move`、`atlas_rover_stop`。
  - 支持 `atlas.mimiclaw.v1` 结构化意图包。
  - 补齐 love、money、cry 表情枚举和基础渲染。
- 新增 `integrations/mimiclaw/spiffs_data/skills/atlas-rover-control.md`，作为后续放入 MimiClaw SPIFFS 的 Atlas Rover 控制 skill。
- V0.5 本地 `idf.py build` 编译通过，固件大小约 `0x110770`。
- 按 Mk.1 简化底盘复审并修正口径：
  - 先采用普通 N20 电机 + DRV8833 + 前万向轮。
  - 不采购编码器/IMU，不做精确里程和角度闭环。
  - `AR1,TURN` 改为方向、速度、持续时间三参数，例如 `AR1,TURN,L,30,350`。
  - MimiClaw skill 明确只能输出短时开环动作，不能要求精确距离/角度。
- 细化底盘控制板方案：
  - Mk.1 底盘板锁定为 Seeed Studio XIAO ESP32C3。
  - DualEye LCD1 Pin10/Pin9 通过 UART 接 XIAO D7/D6。
  - XIAO D2-D5 连接 DRV8833 AIN1/AIN2/BIN1/BIN2。
  - 采购清单补入 XIAO ESP32C3 底盘控制板。
- 完成 XIAO ESP32C3 底盘固件 V0.1：
  - 新增 `firmware/chassis_xiao_esp32c3` ESP-IDF 工程。
  - 使用 UART1：XIAO D7/GPIO20 收 DualEye 命令，D6/GPIO21 回 ACK。
  - 使用 D2-D5 四路 PWM 控制 DRV8833。
  - 只接受 `AR1,STOP`、`AR1,MOVE,F/B,speed,duration_ms`、`AR1,TURN,L/R,speed,duration_ms`。
  - 拒绝旧的无持续时间转向命令，异常命令立即停车并回 `AR1,ACK,ERR`。
  - 独立动作截止看门狗，达到 `duration_ms` 自动停车。

### 仍然不确定

- DualEye 双屏显示后端已接入 GC9A01/LVGL 初始化，但真机旋转、背光和触摸仍需实测校准。
- PSRAM 模式仍需跟真机显示缓冲策略一起验证。
- MimiClaw 语音链路已有 DualEye 接口层，真实 MimiClaw agent loop 尚未合并进同一固件。
- Web 管理页还只是基础嵌入式页面，没有做漂亮 UI、日志面板和 OTA。
- mDNS 暂未接入，当前依赖 SoftAP 地址或串口日志里的局域网 IP。
- API Key 当前存 NVS，但还未启用 NVS 加密，暂不适合放高价值密钥。
- 底盘固件已可构建，真实车轮方向、左右电机极性、PWM 起步阈值和转向时长还需要上车标定。
- 铜丝车架需要真实打样后验证强度、绝缘和可维护性。

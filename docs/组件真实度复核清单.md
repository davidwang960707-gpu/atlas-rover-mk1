# 组件真实度复核清单

日期：2026-06-19

## 分级口径

- `asset_verified`：已用官方 CAD / STEP / STL / GLB 等资产校准外形或关键包围盒。
- `measured`：已按平台原生参数化模型的设计尺寸校准，渲染、碰撞和清单共用同一组尺寸。
- `vendor_verified`：已用厂商规格、数据手册或产品页校准型号和尺寸。
- `source_estimated`：来源能确认组件身份或使用场景，但外形尺寸仍来自资料估算。
- `placeholder`：仅占位，不应进入正式装配评审。

## 本轮升级明细

### Qmini

- `qmini_base_link` 与 10 个腿部 URDF STL 结构件：升级为 `asset_verified`，包围盒来自官方 STL 本地测量并从米转换为毫米。
- `qmini_8010_motor`：升级为 `asset_verified`，官方 STEP 装配中包含 `MOTOR_MDOEL_SIMPLE_UPDATE-8010` 产品/实体名称。
- 已新增 `packages/qmini-open-source/assets/step-index/qmini-v2.0-step-index.json`。OCCT 临时探测能读取 2227 个装配节点和 26 个命名节点，其中包括 `顶部提手`、`Go1_BatteryCharger_V0`、`Go1电池盖装配体简化` 等；但该解析路径没有输出三角顶点，不能直接量取外观件包围盒。
- Qmini 外观细节仍保留 `source_estimated`，但状态已细分：顶部把手为“找到 STEP 命名节点但未网格化”，其他面罩/耳翼/线缆环/侧圆盖为“未找到直接命名网格，需 CAD 拆分”。

### SmallPond D-BOT

- `dbot_esp32s3_wroom`：升级为 `vendor_verified`，采用 ESP32-S3-WROOM-1U 18.0 x 19.2 x 3.2 mm 包络。
- `dbot_gc9a01_round_lcd`：升级为 `vendor_verified`，采用 1.28 寸 GC9A01 圆屏面板尺寸。
- `dbot_mt6701_encoder`：升级为 `vendor_verified`，从估算编码器板改为 MT6701CT SOP-8 编码器芯片包络。
- 已新增 `packages/smallpond-dbot/assets/external_asset_intake.json`。当前匿名探测结果：OSHWHub 命令行访问返回 WAF 418；MakerWorld 下载页返回 Cloudflare 403；OSHWHub 浏览索引可见 3D 模型区暂无数据、附件仅列 BOM。
- PCB 与打印件仅能从 D-BOT README 确认角色和名称，外形继续等待登录态导出、作者提供文件或平台资产上传。

### 桌面人形结构 Starter

- 14 个平台原生 `humanoid_structure` 组件升级为 `measured`，表示它们的尺寸由平台参数化模型直接定义。
- `dhs_micro_servo_9g`：升级为 `vendor_verified`，并切换到 `servo_actuator` 参数化舵机模型。
- `dhs_servo_driver_16ch`：升级为 `vendor_verified`，采用 Adafruit PCA9685 16 路舵机驱动板尺寸。

### Atlas Rover Mk1

- `boost_5v_high_current` 改名为 `MT3608 2A 可调升压模块`，升级为 `vendor_verified`。
- 同步修正输出电流和规则：该模块用于 5V 逻辑/附件供电，不再伪装为 4A 电机电源。

## 后续复核队列

1. D-BOT：使用登录态接入 OSHWHub PCB/Gerber，量取主控板、驱动板、屏幕板、电池板轮廓和孔位。
2. D-BOT：通过 MakerWorld 登录态或作者文件下载 STL/3MF，计算 5 个打印件包围盒并绑定真实资产预览。
3. Qmini：用 FreeCAD/OCCT 服务端作业拆分官方 STEP 外观层零件，优先导出 `顶部提手`，再处理面罩、耳翼、线缆环、侧圆盖。
4. Atlas Rover Mk1：选定 N20、64T 齿轮轮、DRV8833、18650 电池盒、黄铜棒/铜柱供应商，再升级为 `vendor_verified` 或 `measured`。
5. 桌面人形：在用户基于模板创建项目时，要求从标准件库选择具体控制板、电池、IMU 和紧固件规格。

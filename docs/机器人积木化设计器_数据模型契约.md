# 机器人积木化设计器数据模型契约

版本：2026.06

这份契约用于约束机器人组件包、布局、连接图和制造输出的核心数据。机器可读版本位于 `simulator_web/rover_builder/schemas/robot-package-contract.v1.json`。

## 包结构

- `catalog.json`：平台可发现的组件包目录。
- `package.json`：组件包 manifest，声明包 id、版本、默认方案、数据模型版本和文件入口。
- `components.json`：组件类型库，包含几何、类别、接口、BOM、来源和安装信息。
- `layouts/default.json`：示例布局，包含实例 pose 和连接图。
- `rules.json`：规则集，描述数量、连接、安全和制造复核要求。

## 核心对象

- 组件类型：`id`、`category`、`label`、`geometry`、`ports`、`bom`、`model`。
- 安装点：位于 `defaults.base_plate.mount_points`，包含 `id`、`kind`、`x`、`y`，可选 `fastener`、`role`、`slot`。
- 布局实例：`id`、`componentId`、`pose`、`role`，可选 `side`、`axle`。
- 接口端点：`instanceId.portId`，例如 `drv-front.outputs`。
- 连接：`id`、`from`、`to`、`kind`，连接两端必须存在且端口类型兼容。

## 端口兼容

端口兼容以机器可读契约里的 `endpointRules` 为准。运行时连接建议、导入预检和命令行验证器都读取同一套规则。

- `motor_output` 只能连接 `dc_motor_2wire`。
- `gpio_pwm` / `gpio_signal` / `single_wire_data_optional` 可连接 `logic_input` / `single_wire_data`。
- `uart_3v3` 只连接 `uart_3v3`。
- `speaker_differential` 只连接 `speaker_differential`。
- `battery_power` / `battery_input` / `5v_power_bus` / `5v` 可形成供电链路，并可连接 `motor_power`。
- `usb` 只作为空间和可达性接口，不自动生成线束连接。

## 几何与 BOM

几何尺寸必须能归一化为可计算包围盒，供 3D 渲染、吸附、碰撞、越界和制造预览共用。

- 板卡/盒体使用 `length`、`width`、`thickness` 或 `height`。
- 圆柱/灯环使用 `outer_diameter` 或 `outer_diameter_default`，并提供厚度。
- 电机使用 `body_length`、`body_width`、`body_height`，可附加齿箱和轴信息。
- 程序化结构件必须提供 `span_x`、`span_y`、`target_height`。

组件外观资产使用 `model` 描述。该字段用于区分真实还原程度和扩展资产引用，不参与 BOM 数量计算。

- `model.fidelity`：模型还原度，取值为 `measured`、`asset_verified`、`vendor_verified`、`source_estimated`、`placeholder`。
- `model.visual_profile`：当前渲染器使用的过程化外观 profile，例如 `n20_ga12_metal_gearmotor_with_d_shaft`。
- `model.asset_refs`：可选资产引用，支持 `reference`、`reference_image`、`glb`、`gltf`、`step`、`stp`、`stl`。当前 MVP 先展示和随包导出，后续渲染器可按 URI 加载 GLB，制造侧可按 STEP/STL 引用。

类别用于组件库分门别类管理。当前契约支持 `core`、`control`、`motion`、`driver`、`power`、`sensor`、`comms`、`actuator`、`lighting`、`audio`、`fastener`、`structure`。

BOM 字段需要机器可计算，不能把自然语言混入数量字段。

- `bom.quantity`：正数，用于计算、排序和导出。
- `bom.unit`：可选，取值来自契约 `bomUnits`，例如 `piece`、`set`、`pack`、`meter`、`mm`。
- `bom.quantity_note`：可选，用于说明裁切、套件或按图加工等人工语义。
- `bom.required`：取值来自契约 `bomRequiredValues`，例如 `owned`、`required`、`recommended`、`optional`。
- `sources`：组件必须提供来源、规格依据或明确的占位复核说明。

## 当前落地

- 组件管理页面展示每个包的模型健康度。
- 右侧组件详情展示包数据模型摘要和问题列表。
- 吸附点、底板孔位图和制造预览读取组件包里的 `mount_points`。
- 接口建议、连接图校验和导入预检读取契约里的 `endpointRules`。
- `tools/validate_robot_designer_packages.mjs` 会校验 schema、类别、几何、BOM、来源、安装点、端口、实例、连接和兼容性。
- 布局导出会携带 `dataModel` 信息，为后续版本迁移做准备。

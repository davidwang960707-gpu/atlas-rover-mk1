# 机器人积木化设计器 · 产品经理审阅（平台首页与社区体验）

> 目标：让平台“第一次打开即懂”，让新用户能在 10 分钟内完成一次可评审的机器人方案创建。

## 1. 产品定位

**一句话定位：** 面向机器人爱好者与小团队的“组件化机器人概念验证平台”，提供从灵感到可复用工程产物的一站式主链路。

**边界：**
- 不是通用 CAD 替代品，核心优势是“组件库 + 规则 + 工程产物输出”。
- 不是仅展示页面，强调“能装配、能校验、能导出、能复用”。
- 第一个示例是 `Atlas Rover Mk1`，但默认导向是通用平台入口，不是该案例入口。

## 2. 用户路径

### 新用户（首次进入）
1. **落地页**：看懂自己是谁、今天能做什么。
2. **3 秒决策**：从零 / 从模板 / 组件库 入口之一进入工作流。
3. **主链路引导**：拖入组件 → 对齐安装点 → 实时风险提示。
4. **交付验证**：在右侧看 BOM/规则/步骤并导出。
5. **社区传播**：生成分享链接或发布组件包。

### 创作者 / 开发者
1. 打开模板或现有项目。
2. 按标准件构建主结构，优先使用组件参数与安装规则。
3. 用规则面板修正装配与电气风险。
4. 导出制造包并形成模板化复用策略。

### 团队协作路径
1. 使用项目与组件包进行版本分支与复用。
2. 在社区/分享中心寻找可复用方案。
3. 通过工作区权限控制资产与项目访问边界。

## 3. 页面结构

### 首页（landing）
- **品牌 + 价值宣言区**：第一屏必须回答“我是谁、我解决什么、我现在能做什么”。
- **10 分钟主路径条**：将四步流程可视化。
- **指标区**：社区规模、项目数、组件包、真实资产可见度。
- **社区与用户空间摘要**：降低信任门槛，快速显示平台活性。
- **新手场景卡**：按角色给出首选入口。

### 工作台（home）
- 保留 `stat-grid` 与可视化指标。
- 新增“第一次创建流程区块”，避免用户直接陷入大量卡片噪音。
- 关键任务区：最近项目、模板入口、社区入口、分享、资产与工作区管理。

### 社区页（community）
- 保持“项目/组件包/挑战/创客/动态”五类入口。
- 默认展示项应聚焦“可复用资产”，不是空列表。
- 所有卡片统一展示“路径动作”（查看、打开、参与）与可见性。

### 资源页（resources/components/assets/packages）
- 保持“统一的类卡片视图 + 搜索过滤”。
- 统一卡片语义：名称、规格、状态、动作。

## 4. 视觉语言

- **主色**：沿用当前暗色工业底色，不要再加入过强花哨色。
- **语气**：强调真实感与工程感，减少“游戏化”视觉。
- **层级**：每个模块必须有`manager-panel`级别边界与统一边框/间距。
- **按钮策略**：主动作为 1 个（如创建/进入），次动作控制在 2~3 个，避免同屏按钮过载。
- **信息密度控制**：长页面分片（hero / 路径 / 指标 / 列表）逐层出现。

## 5. 第一屏吸引力

- 头图文案改为：先说**结果**（工程可复用），再说**边界**（不是单模型浏览）。
- CTA 建议固定为：
  1. 进入工作台（主目标）
  2. 从模板开始
  3. 从零开始
  4. 打开组件库
  5. 进入社区
- 主路径卡与社区卡都应展示“下一步按钮”，减少猜测成本。

## 6. 社区感

- 落地页默认提供“真实社区入口预览”，避免只剩本地静态占位。
- 主页中应将“协作与治理”和“社区”明确区分：
  - 协作与治理：权限、分享、工作区、审核；
  - 社区：公开项目、组件包、挑战、动态。
- 在导航中保留清晰路径（导航标签可同时突出“协作与治理”和“社区”）。

## 7. 功能落点

### 当前已对齐
- 落地页路径与主链路入口已补齐。
- 首页新增新手启动路线区块。
- 顶部导航补充“社区”入口，首页/社区文案更偏工程叙事。
- 页面结构加入统一的卡片化节律与主路径说明。

### 下一版优先级（建议）
1. `community` 与 `collaboration` 页面增加入口互斥说明文案，避免用户困惑。
2. 落地页展示的案例卡片默认添加“查看模板 / 下载布局”的统一动作。
3. 增加“本次会话建议下一步”小提示条（例如“继续到模板”/“继续到组件库”）。
4. 为首屏加入轻量新手成功率埋点（创建项目、进入设计器、导出一次 BOM）。

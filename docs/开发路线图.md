# Atlas Rover Mk.1 开发路线图

这不是承诺书，是一张不断修正的地图。

## Milestone 0：资料整理与方向确认

状态：已完成第一轮。

- 制造图纸包 V1.0。
- 采购清单和规格复审。
- 双板方案确认。
- DualEye UART 接线方案确认。
- VS Code / ESP-IDF / Live Preview 开发环境确认。

## Milestone 1：DualEye HMI 跑起来

状态：进行中。

- 已完成 DualEye 固件 V0.5 骨架、配网/API、主题参数、应用页枚举和 MimiClaw 结构化意图入口。
- 已完成 Web 表情预览。
- 已接入 Waveshare 官方同款 GC9A01/LVGL 双屏 LCD 后端。
- 待接入触摸页面切换。
- 待接入音频输入/输出。
- 待真机校准左右眼旋转、背光和页面细节。

验收标准：

- 真机双屏能显示左右眼。
- 语音/触摸事件能切换表情。
- 串口监视器可看到 UART 协议和状态日志。

## Milestone 2：配网、API 与管理界面

状态：V0.2 基础骨架已实现，真机体验和完整调试能力待完善。

- 已实现首次启动 SoftAP：`AtlasRover-XXXX` / `192.168.4.1`。
- 已实现 STA 优先连接和失败回落 APSTA。
- 已实现 NVS 保存 Wi-Fi、LLM 模式、Base URL、Model、API Key、安全限制。
- 已实现基础 Web 管理页：状态、Wi-Fi、LLM/API、安全、STOP、短时移动、文本意图测试。
- 已实现配对码：STOP 不需要配对码，移动和配置修改需要配对码。
- 已实现 API Key 不写源码、不进 GitHub、不打印日志。
- 待实现 mDNS 或更友好的设备发现。
- 待完善电脑端日志、UART TX/RX、LLM 工具调用观察。
- 待完善手机端表情/页面切换和更好的触摸交互。

验收标准：

- 新烧录设备可用手机完成配网。
- 可在 Web 页面配置/清除大模型 API。
- STOP 按钮常驻且能立即下发 `AR1,STOP`。
- 运动控制受限速、短时和底盘在线状态保护。
- 配置流程在真机上验证通过，并补截图/踩坑记录。

## Milestone 3：底盘板固件

状态：待开始。

- 底盘控制板已锁定：Seeed Studio XIAO ESP32C3。
- 实现普通 N20 + DRV8833 开环短时差速控制。
- 实现限速、加减速、超时停车。
- 实现 `AR1,ACK,OK/BUSY/ERR`。
- 验证前进、后退、左转、右转、停止。

验收标准：

- DualEye 发送 `AR1,MOVE/F/TURN/STOP` 后，底盘板按速度和持续时间响应。
- 丢失 UART 指令或异常时自动停车。
- 电机供电不经过 DualEye。

## Milestone 4：机械打样

状态：待打样。

- 铜丝/黄铜车架实际弯折和焊接。
- 底盘、电池、电机、DualEye 的安装位置验证。
- 绝缘、重心、维护空间检查。
- 线束固定和接口防松。

验收标准：

- 车架能承载电池、电路板和双眼头部。
- 电机、轮子、万向轮没有干涉。
- 拆装维护不需要破坏铜丝主体结构。

## Milestone 5：语音与 MimiClaw

状态：DualEye 侧 MimiClaw 结构化意图入口已开始落地。

- 麦克风收音。
- 唤醒/聆听/思考/说话状态。
- 支持三种运行模式：LLM 关闭、外部宿主/调试桥、端侧 MimiClaw。
- 语义意图转 `atlas_voice_event_t`。
- MimiClaw 侧预置 Atlas Rover control skill，并通过 tool-call 或 `atlas.mimiclaw.v1` 意图提交给 DualEye。
- DualEye 侧通过 `/api/mimiclaw/intent` 解析和安全裁剪。
- 移动指令需要确认安全边界。
- TTS 音量驱动 speaking 表情。

验收标准：

- “前进/后退/左转/右转/停止”能稳定映射到底盘指令。
- 非移动类对话不会误触发电机。
- 错误和拒绝场景有明确表情反馈。

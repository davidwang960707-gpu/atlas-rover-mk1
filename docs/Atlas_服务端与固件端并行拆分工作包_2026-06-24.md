# Atlas 服务端与固件端并行拆分工作包

日期：2026-06-24  
目标：冻结当前可运行基线，然后用两个 worktree / 会话并行推进服务端和固件端平台化拆分。

## 1. 基线原则

本轮 checkpoint 只覆盖 Atlas Brain、DualEye 固件、协议 specs、项目文档和 DualEye 必要资源。

暂不纳入：

- `tmp/`
- `outputs/`
- `var/`
- `simulator_web/rover_builder/`
- `atlas_sense_box_mk1/`
- `智能台灯设计图/`

原因：这些目录属于运行缓存、导出物、其他产品线或大体积外部资产，不应该污染这次服务端/固件端拆分基线。

## 2. 两条工作线

### A. 服务端工作线

建议分支：

```bash
codex/atlas-brain-platform-split
```

职责：

- 继续瘦身 `tools/atlas_brain_server.py`。
- 拆出 `atlas_brain_audio_routes.py`。
- 拆出 `atlas_brain_ota.py`。
- 拆出 `atlas_brain_weather.py`。
- 拆出 `atlas_brain_routes.py` 或路由注册层。
- 保持 `/health`、`/api/tools/list`、`/api/tools/call`、`/mcp/tools/list`、`/ws/brain`、`/ws/audio`、`/turn/audio`、`/device/audio/wav` 路径不变。

禁止直接修改：

- `firmware/dualeye/**`
- `assets/dualeye_sdcard_v0_1/**`，除非服务端验收必须同步资源 manifest。

每轮验收：

```bash
python3 -m py_compile tools/atlas_brain_server.py tools/atlas_brain_audio.py tools/atlas_brain_devices.py tools/atlas_brain_tools.py tools/atlas_brain_core.py tools/atlas_brain_providers.py tools/atlas_brain_runtime.py tools/atlas_web_ui.py
python3 tools/atlas_brain_server.py --dry-run --host 127.0.0.1 --port 8799
python3 tools/check_atlas_preflash.py --brain-url http://127.0.0.1:8799 --skip-dualeye
```

### B. 固件端工作线

建议分支：

```bash
codex/atlas-firmware-common-split
```

职责：

- 梳理 `firmware/dualeye/main` 的模块边界。
- 标记并逐步拆出 common runtime/audio/brain client/app/board 边界。
- 保留 public wrapper，避免一次性改坏页面、音频、Brain 连接。
- 固件端优先保证离线显示、Brain 离线降级、状态页、自检和 OPUS 真流入口。

禁止直接修改：

- `tools/atlas_brain_server.py`
- `tools/atlas_brain_audio.py`
- `tools/atlas_brain_devices.py`
- `tools/atlas_brain_tools.py`

允许修改：

- `specs/**`，但必须先在文档中写清楚兼容策略。
- `assets/dualeye_sdcard_v0_1/**`，仅限固件资源加载或 manifest 对齐。

每轮验收：

```bash
cd firmware/dualeye
idf.py build
```

真机联调时再跑：

```bash
python3 tools/check_atlas_preflash.py --brain-url http://127.0.0.1:8787
```

## 3. 共享契约规则

`specs/**` 是两条工作线的共享边界。

规则：

- 能不改 spec 就不改。
- 必须改 spec 时，先写兼容字段，不删旧字段。
- 服务端和固件端都不得依赖未记录在 specs 的隐式字段。
- 协议字段新增后，至少补一个验收点或自检点。

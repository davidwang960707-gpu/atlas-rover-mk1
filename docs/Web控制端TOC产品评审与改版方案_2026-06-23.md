# Atlas Web 控制端 TOC 产品评审与改版方案

日期：2026-06-23

## 产品经理评审结论

上一版 Web 端的核心问题不是视觉风格，而是产品定位错位：它把研发验收、Provider、协议、P2/P3、OPUS、JSON 结果放在首屏，普通用户打开后不知道先做什么，也感受不到“这是我的桌面机器人”。

## 用户视角

目标用户不是工程师，而是刚拿到 Atlas DualEye 的普通用户。用户每天更关心：

- 和机器人说话
- 切换眼睛主题和人格
- 打开时钟、番茄、日历、天气、故事、音乐
- 知道机器人是否在线、是否能听、是否能说

不应该首屏暴露：

- OPUS/P2/P3/P4/P5
- Provider/协议通道/运行时评分
- 大段 JSON
- 烧录验收入口

## 本轮改版原则

1. `/` 是用户日常控制台，不再是工程后台。
2. `/devices/dualeye/app` 与 `/` 保持一致，都是机器人 App。
3. `/admin` 只服务开发者，做平台状态、模型、协议、OTA 和诊断。
4. 高级诊断折叠收起，默认不打扰普通用户。
5. 所有快捷入口都走现有 SkillRegistry，避免 UI 和语音能力割裂。

## 信息架构

### 用户端

- 机器人状态与双眼视觉
- 对话输入与语音输入
- 角色/主题切换
- 应用入口：时钟、番茄、日历、天气、音乐、故事
- 模型/Brain/播报状态
- 折叠高级诊断

### 管理端

- 平台状态
- LLM / ASR / TTS / 天气状态
- 验收报告、运行评分、平台快照
- Provider、协议、工具表、音频流、OTA
- 原始诊断输出

## 下一轮产品目标

- 让设备在线后首屏动态展示真实 DualEye 页面、主题、表情和播报状态
- 把语音对话从“录音按钮”升级为“连续对话会话”
- 为时钟、番茄、日历分别做独立应用详情页
- 把音色、人格和主题做成统一的“性格设置”
- 加一个新手引导：配网、连接 Brain、试说一句话、切换主题

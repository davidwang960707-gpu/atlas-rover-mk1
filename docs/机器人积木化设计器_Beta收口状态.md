# 机器人积木化设计器 Beta 收口状态

本文档用于给产品、设计、开发和评审者快速判断：当前版本是否已经达到“可连续体验、可演示、可继续迭代”的 Beta 标准。

## Beta 定义

Beta 版不是生产 SaaS。它要求主链路完整、平台骨架清楚、数据能持久化到本地 Platform API、核心页面可用、主流程自动化测试通过。

生产上线仍需要真实登录、云数据库、对象存储、部署域名、备份和压测。

## 当前 Beta 主链路

- 从首页查看 Beta 状态。
- 从零创建项目或从模板创建项目。
- 在项目管理里查看项目、版本、分支和制造输出。
- 在组件库和组件包页管理标准件、示例机器人包、私有包、公开包和发布审核。
- 在资产库上传 STL/GLB/STEP，查看预览、版本、授权、处理队列和治理问题。
- 进入设计器做 3D 装配、吸附、碰撞/越界提示、评审和制造输出。
- 在分享中心创建只读/评论链接并撤销。
- 在洞察页查看影响图、风险队列、正式后端准备度和后端契约。

## Beta 发布中心

首页新增 Beta 发布中心，由 `/api/platform/beta-readiness` 提供数据。它会汇总：

- 项目、项目版本和设计文档。
- 模板数量。
- 组件类型和组件包数量。
- 资产、ready 资产、对象存储和处理队列。
- 制造输出归档。
- 分享链接。
- 后端契约表数、worker 队列和审计事件。

首页同时新增 Beta 体验推进台，由 `/api/platform/beta-experience` 提供数据。它把 Beta 后的体验动作拆成：

- 主链路体验顺序：模板创建、设计器评审、资产治理、分享与制造输出。
- 演示数据卫生：识别 UI 回归测试留下的资产、组件包和测试分享，支持一键归档明确测试噪音。
- 生产化推进包：组件真实度扩展、正式后端迁移、个人创客生态。

状态分为：

- `ready`：主链路已达到 Beta 演示条件。
- `review`：核心能力可用，但仍有少量演示风险。
- `blocked`：存在关键阻断项，不建议演示。

## Beta 门禁

当前 Beta 门禁包括：

- 项目管理。
- 模板创建。
- 组件库和组件包。
- 资产上传和真实模型。
- 设计器评审链路。
- 制造输出。
- 分享中心。
- 平台洞察。

## 已具备的平台后端能力

- 本地 Platform API。
- 用户、工作区、成员角色。
- 项目、分支、命名版本、设计文档 revision。
- 组件包、组件包版本、发布审核和公开市场 fork。
- 资产上传、对象存储抽象、签名 URL、版本快照、派生物和处理队列。
- 分享链接和评论能力。
- 制造输出归档。
- 平台影响图。
- 后端契约：Postgres 表、对象存储策略、worker 队列、权限矩阵和迁移 gate。

## 仍需生产化

- 将本地 JSON collections 迁移到 Postgres。
- 将 local-fs 对象存储迁移到 S3/R2/OSS。
- 将本地 asset/export worker 拆成独立队列服务。
- 接入真实 OAuth/OIDC 或邮箱登录。
- 分享链接生产环境只保存 token hash。
- 增加部署、备份、监控、压测和灰度发布流程。

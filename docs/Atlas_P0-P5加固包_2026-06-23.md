# Atlas P0-P5 加固包记录

日期：2026-06-23

版本：`0.14.3-atlas-brain`

## 本轮目标

把固件侧和 Web/Mac Brain 侧的 P0-P5 从“有骨架”推进到“可编译、可验收、可解释失败原因”的状态。验收口径不把未实机验证的常驻 WakeNet/AEC、流式 ASR/TTS 说成完成；但 app OTA、ESP-SR 模型包和资源探针已经进入真实构建链路。

| 优先级 | 结果 |

## 仍需实机确认

- DualEye 烧录后 `/api/status` 与 `/api/status/lite` 都要返回 `0.14.3-atlas-brain`。
- `/api/selftest` 需要 `fail=0`，如果不是 0，先看具体 failed item，不再靠肉眼猜。
- `/api/status/lite` 需要出现 `brain_ws` 字段；host 模式、同 Wi-Fi、Mac Brain 运行时，`brain_ws.connected` 应为 `true`，否则看 `brain_ws.stage` 和 `last_error`。
- 断开 Mac Brain 后触发 `/api/voice/turn` 应返回 `{"error":"brain_offline"}`；设备屏幕仍能切换/保持双眼、时钟、日历和番茄，不应出现黑屏或异常文字页。
- OPUS 真机流需要在 Mac Brain `/acceptance` 页面跑一次 1.8 秒，确认真机 frames、sequence_gaps、payload_len_mismatches。
- `/api/sr/status` 需要返回 `model_load_ok=true`、`wakenet_create_ok=true`、`ready_for_wakenet=true`；如果不是，优先检查是否全量烧录了 `build/srmodels/srmodels.bin`。
- WakeNet 常驻任务和 AEC 仍不是完成项；下一步是实机 AB：只开 WakeNet、只开播放期 mute、验证是否能拿到扬声器参考输入，再决定是否上 AFE AEC。
- app OTA 已可用，但这版分区表和 SR 模型分区变化，首次升级必须 USB 全量烧录；后续只改 app 时才可用 `/api/ota/apply`。
- MiMo Key 只应放在本地 `.atlas-brain.env`，不要写入源码、文档、固件 NVS 导出或串口日志。
